== Enforce Consistent Git Commit Messages

*Gommitlint* footnote:[Dad joke intended - Commitlint, written in Go-lang - it has to be Go-mmitlint, right] validates Git commit messages (and repository state) to help you maintain a quality project history.
It works out-of-the-box with strict sensible (imho) defaults while being configurable.

* **Zero Configuration** - Strict sensible defaults for good commit practices, with Conventional Commits
* **CI/CD Ready** - Multiple output formats for automation


== Available Rules

All rules are enabled by default except `jirareference` and `commitbody`.

=== Message Rules
* **subjectlength** - Limits subject line length (default: 72 chars)
* **subjectcase** - Enforces consistent capitalization (sentence/lower/upper/title)
* **subjectsuffix** - Prevents subjects ending with periods, commas, etc.
* **imperative** - Requires imperative mood ("Add" not "Added")
* **commitbody** - Validates body length and content _(disabled by default)_
* **signoff** - Ensures Developer Certificate of Origin
* **spell** - Checks common spelling (only for English)

=== Format Rules  
* **conventional** - Enforces https://www.conventionalcommits.org[Conventional Commits]
* **jirareference** - Requires JIRA ticket references _(disabled by default)_

=== Security Rules
* **signature** - Verifies GPG/SSH signatures  
* **identity** - Validates signer identity against allowed keys

=== Repository Rules
* **branchahead** - Limits branch divergence from main

== Installation and Quickstart

=== Alternative 1. Download Packages or Binary
Get pre-built binaries from https://github.com/itiquette/gommitlint/releases[releases].

=== Alternative 2. Use Go Install
[source,bash]
----
go install github.com/itiquette/gommitlint/cmd/gommitlint@latest
----

=== Alternative 3. Run in a Container
[source,bash]
----
podman run --rm -v $PWD:/src -w /src ghcr.io/itiquette/gommitlint:latest validate
----

That's it! Gommitlint is now ready to protect your commit quality.

== Usage Examples

=== Basic Validation
[source,bash]
----
# Validate last commit
gommitlint validate

# Validate specific commit
gommitlint validate --git-reference=abc123

# Validate commit range
gommitlint validate --git-reference=main..HEAD
----

=== Git Hooks
[source,bash]
----
# Install commit-msg hook
gommitlint install-hook

# Remove hook
gommitlint remove-hook
----

=== CI/CD Integration
[source,yaml]
----
# GitHub Actions
- name: Validate commits
  run: |
    go install github.com/itiquette/gommitlint/cmd/gommitlint@latest
    gommitlint validate --git-reference=${{ github.event.pull_request.base.sha }}..HEAD
----

== Configuration

Gommitlint works without configuration, having sensible defaults.
To customize, create `.gommitlint.yaml`:

=== Simple Configuration
[source,yaml]
----
gommitlint:
  rules:
    enabled:
      - jirareference    # Enable JIRA validation  
    disabled:
      - spell           # Disable spell checking
  message:
    subject:
      max_length: 50    # Customize subject length
----

See link:docs/usage.adoc[Usage Guide] for complete configuration options.

== For Developers

=== Design Philosophy

Gommitlint tries to follow the following practices:

* **Hexagonal Architecture** - Clean separation of concerns with simplified ports
* **Functional Programming** - Pure functions and immutability throughout
* **Value Semantics** - Predictable behavior without pointer receivers
* **Simple Dependency Injection** - Direct instantiation without containers

Read more in the link:docs/ARCHITECTURE.md[Architecture Documentation].

=== Building from Source
[source,bash]
----
# Clone
git clone https://github.com/itiquette/gommitlint
cd gommitlint

# Build
make build/plain

# Test
make quality

# Install locally
make install
----
