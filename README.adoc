== What can this tool do for you?
*Gommitlint* is a simple CLI-tool for validating Git commit messages and repository state.
At the moment it is in early development.

== Where can you use it?

* As a CLI-tool for pre-commit validation
* In a CI/CD-pipeline to enforce team standards
* As a Git hook for automatic validation

=== Available validation rules:

TE
==== Commit Message Rules

* *CommitsAhead* - Limits how far a branch can diverge from a reference branch
* *ConventionalCommit* - Enforces https://www.conventionalcommits.org[Conventional Commits] format with configurable types and scopes
* *ImperativeVerb* - Validates that commit messages begin with a verb in the imperative mood
* *JiraReference* - Verifies commits reference valid Jira issue keys in a consistent format
* *SubjectCase* - Enforces consistent capitalization in commit subjects
* *SubjectSuffix* - Prevents commit subjects from ending with specified characters
* *SubjectLength* - Limits commit subject line length for readability (default: 100 chars)
* *SignOff* - Ensures commits include a Developer Certificate of Origin sign-off line
* *Spell* - Catches common spelling mistakes with locale-specific dictionaries
* *Signature* - Verifies commits have a cryptographic signature (GPG or SSH)
* *SignedIdentity* - Validates signatures against trusted keys with full cryptographic verification

== Getting Started
TODO
//1. Check out the link:docs/usage.adoc[Usage Guide] for a quick start.
// 2. The link:INSTALL.adoc[Installation Guide] will get you up and running.

  gommitlint:
    subject:
      case: "sentence"           # Options: sentence, lower, upper
      max_length: 72             # Maximum length of subject line
      require_imperative: true   # Whether subjects should use imperative mood
      disallowed_suffixes:       # List of disallowed ending characters
        - "."
        - ","
        - ";"

    body:
      required: false            # Whether a commit body is required
      min_length: 10             # Minimum body length
      minimum_lines: 3           # Minimum number of lines in body
      allow_sign_off_only: true  # Whether body can only contain sign-off info

    conventional:
      required: true             # Whether conventional commits are required
      require_scope: false       # Whether scope is required in conventional format
      types:                     # Allowed commit types
        - "feat"
        - "fix"
        - "docs"
        - "style"
        - "refactor"
        - "perf"
        - "test"
        - "build"
        - "ci"
        - "chore"
        - "revert"
      scopes: []                 # Allowed scopes (empty means any scope is allowed)
      allow_breaking_changes: true # Whether breaking changes are allowed
      max_description_length: 72 # Maximum length of description

    rules:
      enabled:                   # Explicitly enabled rules
        - SubjectLength
        - SubjectCase
        - SubjectSuffix
        - ConventionalCommit
        - ImperativeVerb
        - Signature
        - SignOff
        - Spell
        - JiraReference
        - CommitsAhead
      disabled:                  # Explicitly disabled rules
        - CommitBody           # Disabled by default

    security:
      sign_off_required: true    # Whether sign-off is required
      gpg_required: false        # Whether GPG signature is required
      key_directory: ""          # Path to key directory
      allowed_signature_types:   # Allowed signature types
        - "GPG"
        - "SSH"
      allowed_keyrings: []       # List of allowed keyrings
      allowed_identities: []     # List of allowed identities
      allow_multiple_sign_offs: true # Whether multiple sign-offs are allowed

    repository:
      path: ""                   # Path to repository
      reference_branch: "main"   # Reference branch for CommitsAhead rule
      max_commits_ahead: 5       # Maximum commits ahead of reference
      max_history_days: 365      # Maximum days to look back in history
      output_format: "text"      # Output format
      ignore_merge_commits: true # Whether to ignore merge commits

    output:
      format: "text"             # Output format: text, json, github, gitlab
      verbose: false             # Whether to show verbose output
      quiet: false               # Whether to suppress output
      color: true                # Whether to colorize output

    spell_check:
      language: "en-US"          # Spell check language
      ignore_case: false         # Whether to ignore case in spell checking
      custom_dictionary:         # Custom words to allow
        - "gommitlint"
        - "zerolog"

    jira:
      pattern: "[A-Z]+-\\d+"     # Pattern for JIRA ticket references
      projects: []               # List of allowed JIRA projects
      body_ref: false            # Whether JIRA references can be in body

== For developers

=== Design Philosophy

Gommitlint is built using:

* *Hexagonal Architecture* for clean separation of concerns
* *Functional Programming* principles for predictable and testable code
* *Value Semantics* for improved safety and reasoning

See our link:docs/ARCHITECTURE.md[Architecture Documentation] and link:docs/FUNCTIONAL_PROGRAMMING.md[Functional Programming Guide] for more details.
