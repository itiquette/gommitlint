= Gommitlint Usage Guide

== Basic Usage

=== Validate Commits

[source,bash]
----
# Validate last commit
gommitlint validate

# Validate specific commit
gommitlint validate --git-reference=abc123

# Validate commit range
gommitlint validate --git-reference=main..HEAD

# Validate with verbose output
gommitlint validate -v
----

=== Git Hooks

[source,bash]
----
# Install commit-msg hook
gommitlint install-hook

# Remove hook
gommitlint remove-hook
----

== Configuration

=== Default Behavior

Gommitlint works without configuration. By default:

* Most rules are **enabled** (like subject length, imperative mood)
* `jirareference` and `commitbody` are **disabled by default**
* Configuration file is optional. If you don't like the defaults, configure it to your liking.

=== Simple Configuration

Create `.gommitlint.yaml` to customize behavior:

[source,yaml]
----
gommitlint:
  rules:
    enabled:
      - jirareference    # Enable JIRA validation
    disabled:
      - imperative       # Disable imperative mood check
----

=== Rule Priority System

1. **Explicitly enabled** rules always run (highest priority)
2. **Explicitly disabled** rules are skipped
3. **Default behavior** applies if not specified

[source,yaml]
----
gommitlint:
  rules:
    enabled:
      - commitbody      # Force commitbody check to run
      - jirareference   # Override default-disabled
    disabled:
      - subjectlength   # Disable even though default-enabled
----

=== Full Configuration Example

[source,yaml]
----
gommitlint:
  # Message format rules
  message:
    subject:
      max_length: 72           # Maximum subject length
      case: "sentence"         # sentence|lower|upper|title
      forbid_endings: ["."]    # No period at end
    body:
      min_length: 10          # Minimum body length
      require_signoff: true   # Require Signed-off-by

  # Conventional commits
  conventional:
    require_scope: false      # Make scope optional
    types:                    # Allowed types
      - feat
      - fix
      - docs
      - style
      - refactor
      - perf
      - test
      - build
      - ci
      - chore
      - revert

  # Organization-specific
  jira:
    projects: ["PROJ", "TEAM"]  # Required JIRA prefixes

  # Repository rules  
  repo:
    branch: main               # Target branch
    max_commits_ahead: 10      # Max commits ahead

  # Output format
  output: text                 # text|json|github|gitlab
----

== Available Rules

[cols="1,3,1", options="header"]
|===
|Rule Name |Description |Default

|`subjectlength`
|Validates subject line length
|✓ Enabled

|`subjectcase`
|Validates subject line casing
|✓ Enabled

|`subjectsuffix`
|Forbids certain endings (like periods)
|✓ Enabled

|`imperative`
|Requires imperative mood
|✓ Enabled

|`commitbody`
|Validates commit body requirements
|✗ Disabled

|`conventional`
|Enforces conventional commit format
|✓ Enabled

|`jirareference`
|Requires JIRA ticket reference
|✗ Disabled

|`signoff`
|Requires Signed-off-by line
|✓ Enabled

|`signature`
|Validates GPG/SSH signatures
|✓ Enabled

|`identity`
|Validates committer identity
|✓ Enabled

|`spell`
|Spell-checks commit messages
|✓ Enabled

|`branchahead`
|Checks commits ahead of target branch
|✓ Enabled
|===

== Output Formats Examples

=== Text (Default)
[source,bash]
----
$ gommitlint validate
✓ SubjectLength     Subject is 43 characters (max: 72)
✓ Imperative        Starts with imperative verb
✓ SubjectCase       Sentence case format
✗ CommitBody        Body too short (5 chars, min: 10)
----

=== JSON
[source,bash]
----
$ gommitlint validate --output=json
{
  "commit": "abc123",
  "valid": false,
  "errors": [{
    "rule": "commitbody",
    "code": "body_too_short",
    "message": "Body too short (5 chars, min: 10)"
  }]
}
----

=== GitHub Actions Output
[source,bash]
----
$ gommitlint validate --output=github
::error file=commit.txt,line=3::Body too short (5 chars, min: 10)
----

== Common Workflows

=== CI/CD Integration

[source,yaml]
----
# GitHub Actions
- name: Validate commits
  run: |
    go install github.com/itiquette/gommitlint/cmd/gommitlint@latest
    gommitlint validate --git-reference=${{ github.event.pull_request.base.sha }}..HEAD
----

=== Pre-commit Hook

[source,yaml]
----
# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: gommitlint
        name: Validate commit message
        entry: gommitlint validate --git-reference=HEAD
        language: system
        always_run: true
        pass_filenames: false
----

=== Validate Specific Rules Only

[source,bash]
----
# Only run specific rules
$ gommitlint validate --rules=subjectlength,imperative

# Skip specific rules  
$ gommitlint validate --skip-rules=spell,jirareference
----

== Troubleshooting

=== Debug Output
[source,bash]
----
# Show detailed debug information
$ gommitlint validate -v --debug
----

=== Common Issues

**Rule not running?**
Check if it's in the default-disabled list (`jirareference`, `commitbody`) and explicitly enable it.

**Configuration not loading?**
Gommitlint looks for `.gommitlint.yaml` or `.gommitlint.yml` in the current directory.

**Unexpected validation results?**
Use `-v` flag to see which rules are active and why they pass/fail.

----

  message:

      max_length: 50    # GitHub's recommendation
  rules:
    enabled:
      - commitbody      # Require detailed explanations
    disabled:
      - jirareference   # No JIRA in open source
----

----

  rules:
    enabled:

  jira:

  signing:
